01 · The problem

Why we're rebuilding the data layer

The Demand Generation Dashboard — the custom-built dashboard embedded in your GHL account — lost its automated data feed when Windsor.ai was discontinued due to a billing restriction. The remaining data connection will be lost in a few days, once the Windsor.ai trial account ends.

This project keeps the dashboard exactly as you approved it — same layout, same sections, same metrics from the June revision round — and replaces only the plumbing underneath. Claude Cowork becomes the refresh engine: on a weekday schedule (and on demand), it pulls current data through Zapier from GHL and Google Analytics, regenerates the dashboard, and publishes it. You keep using the same dashboard in the same place; only the numbers get fresher.

02 · How refreshes work

Scheduled, timestamped, and honest about freshness

Each refresh regenerates the dashboard file and publishes it to the page your GHL embed already points at — so updates appear automatically, with no change to how you access it. Refreshes run on a weekday-morning schedule from the Rogers Hollow side, plus on demand whenever you need current numbers.

Two safeguards are built into every version of the page:

  • A visible "Data as of [timestamp]" label, so it's always clear how current the numbers are.
  • A stale-data banner — if any source fails to pull, the dashboard says so rather than silently presenting old numbers as new.

05 · Architecture

Via Zapier connector — sessions · events · channels · landing pages
→
Claude Cowork
Scheduled task pulls both sources and regenerates the dashboard file with fresh data
→
Your GHL Dashboard
Published to the hosted page your embed points at — updates appear automatically

Data is written into the dashboard at generation time, so every published version is a complete, self-contained snapshot. If historical trend views ever become a requirement, a lightweight data store can be added in a later phase — noted, not built now.

06 · Zapier subscription

Plan sizing and cost

Zapier meets the billing requirement that ruled out Windsor.ai, and this build uses your existing Zapier subscription — the goal is to stay within the current budget with no new or unexpected subscription costs. In this architecture, only Google Analytics flows through Zapier (GHL connects directly), which keeps usage low. Each GA4 data pull consumes two "tasks" from the plan's monthly allowance; failed pulls are free. If the allowance were ever exhausted mid-month, the dashboard's GA4 section simply shows its stale-data banner until the allowance resets — GHL sections keep refreshing normally, and nothing fails silently.

09 · Continuity & long-term maintenance

Built so you're never dependent on one person

The setup is designed for ownership: every credential and asset is provisioned under Coldwell-controlled accounts, operated on delegated access. If that access ever ends, you revoke it — nothing needs rebuilding, and nothing leaves with a person.

  • Accounts you own: the Zapier subscription (your existing account), the Google account connected to Analytics, the GHL access token issued from your own sub-account (revocable by you anytime), and the repository holding the dashboard code.
  • Documentation that stays: the full operating runbook — setup from zero, refresh instructions, and a failure playbook — is stored version-controlled alongside the dashboard code itself.
  • Full handoff path: if Coldwell Energy has (or creates) its own Claude account, the entire refresh operation can be recreated under your account and handed to your internal team — roughly an hour of setup following the runbook, plus a supervised refresh as handoff training.
  • Graceful degradation: if refreshes ever pause, no data is lost — GHL and Google Analytics remain the systems of record. The dashboard shows its last-updated timestamp and catches up on the next run.
  • Ongoing checks: the "Data as of" stamp is a health check anyone on your team can glance at; plus a monthly Zapier usage look and a quarterly credential review.

Spike & gates

  • Configure the Zapier connector; connect GHL and Google Analytics.
  • Prove every dashboard metric is retrievable — GHL opportunities, form submissions, won-opps fields, and each GA4 metric with date-range parameters.
  • Measure actual tasks-per-refresh and fit-check against your existing Zapier plan allowance.
  • Verify Google Ads and LinkedIn Ads reporting connections for the Ad Analytics section (Section 07).
  • Confirm the publishing path to the dashboard's hosted page.

GATE — all metrics retrievable + usage within plan budget. No build until cleared.

1–2 days

Data pipeline

  • Full data pull for a known date range; GHL numbers reconciled against the GHL interface, GA4 numbers against the GA4 interface.
  • Per-source error handling wired to the stale-data banner.

1 day

Dashboard regeneration

  • The approved dashboard becomes a template populated fresh on every run; the new Web Analytics section is built in.
  • Side-by-side regression check against the current dashboard on Sections 0–3.

  • Weekday-morning schedule plus on-demand trigger; publishing step wired end-to-end.
